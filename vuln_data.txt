【SQL注入漏洞 (SQL Injection)】
SQL注入是一种代码注入技术，攻击者通过在输入字段中插入恶意SQL代码，篡改原本的SQL查询语句，从而非法获取、修改或删除数据库中的数据。
危险等级：高危
OWASP分类：A03:2021 - 注入
常见场景：登录表单、搜索框、URL参数、API请求参数
攻击案例：在登录框输入 admin' OR '1'='1 可能绕过身份验证
防御方法：使用参数化查询（Prepared Statement）、ORM框架、严格输入验证、最小数据库权限原则、定期进行代码安全审计

【XSS跨站脚本攻击 (Cross-Site Scripting)】
XSS攻击是指攻击者在网页中注入恶意脚本，当其他用户浏览该网页时，恶意脚本会在用户浏览器中执行，可能窃取Cookie、会话信息或进行钓鱼攻击。
危险等级：高危
OWASP分类：A03:2021 - 注入
常见场景：评论区、用户资料展示、搜索结果展示、富文本编辑器
攻击类型：存储型XSS（脚本存入数据库）、反射型XSS（通过URL参数触发）、DOM型XSS（前端JS处理不当）
防御方法：对输出进行HTML转义、设置Content-Security-Policy（CSP）、使用HttpOnly Cookie、对用户输入进行白名单过滤

【CSRF跨站请求伪造 (Cross-Site Request Forgery)】
CSRF攻击是指攻击者诱导已登录用户在不知情的情况下，向目标网站发起非本意的请求，利用用户的登录状态执行恶意操作。
危险等级：中高危
OWASP分类：A01:2021 - 失效的访问控制
常见场景：转账操作、修改密码、发布内容、删除数据等状态变更类请求
防御方法：使用CSRF Token验证、验证Referer/Origin头、设置SameSite Cookie属性、关键操作要求二次确认

【文件上传漏洞 (Unrestricted File Upload)】
文件上传漏洞是指Web应用未对用户上传的文件进行严格校验，导致攻击者可以上传恶意文件（如webshell）并在服务器上执行任意代码。
危险等级：高危
OWASP分类：A04:2021 - 不安全设计
常见场景：头像上传、文档上传、图片上传功能、文件管理系统
防御方法：白名单校验文件类型（而非黑名单）、重命名上传文件、隔离存储目录、上传目录禁止脚本执行权限、文件内容二次检测

【失效的身份认证 (Broken Authentication)】
失效的身份认证是指系统在用户身份验证、会话管理方面存在缺陷，攻击者可借此冒充其他用户身份。
危险等级：高危
OWASP分类：A07:2021 - 身份认证错误
常见场景：弱密码策略、会话ID未及时失效、未限制登录失败次数、密码明文传输
防御方法：实施多因素认证（MFA）、强密码策略、登录失败次数限制与锁定、会话超时与安全的Session管理、密码加盐哈希存储

【安全配置错误 (Security Misconfiguration)】
安全配置错误是指系统、框架、服务器或应用在部署时使用了不安全的默认配置、暴露了不必要的功能或权限过大。
危险等级：中高危
OWASP分类：A05:2021 - 安全配置错误
常见场景：默认账号密码未修改、调试模式在生产环境开启、目录列表暴露、不必要的端口开放、错误信息泄露堆栈细节
防御方法：建立标准化的安全配置基线、关闭生产环境调试信息、定期安全扫描与配置审查、最小化暴露面

【失效的访问控制 (Broken Access Control)】
失效的访问控制是指系统未能正确限制已认证用户的操作权限，导致用户可以访问或操作超出其权限范围的资源。
危险等级：高危
OWASP分类：A01:2021 - 失效的访问控制
常见场景：通过修改URL中的ID直接访问他人数据（越权访问）、普通用户调用管理员接口、水平越权与垂直越权
防御方法：服务端进行权限二次校验、采用基于角色的访问控制（RBAC）、默认拒绝原则、记录并监控异常访问行为

【不安全的反序列化 (Insecure Deserialization)】
不安全的反序列化是指应用程序在反序列化不可信数据时，未做充分校验，攻击者可构造恶意序列化数据执行任意代码。
危险等级：高危
OWASP分类：A08:2021 - 软件和数据完整性故障
常见场景：Java/PHP/Python等语言的对象序列化与反序列化接口、Cookie中存储序列化对象
防御方法：避免反序列化不可信数据、使用数字签名验证数据完整性、采用安全的序列化格式（如JSON替代原生对象序列化）、限制反序列化的类范围

【使用含已知漏洞的组件 (Vulnerable and Outdated Components)】
该风险是指应用使用了包含已知安全漏洞的第三方库、框架或组件，攻击者可利用公开的漏洞信息进行攻击。
危险等级：中高危
OWASP分类：A06:2021 - 易受攻击和过时的组件
常见场景：长期未更新的开源依赖库、过时的服务器软件版本、未及时修补的CMS插件
防御方法：建立软件成分清单（SBOM）、定期扫描依赖漏洞（如使用Snyk、Dependabot）、及时更新补丁、移除不再使用的依赖

【服务端请求伪造 (Server-Side Request Forgery, SSRF)】
SSRF是指攻击者诱导服务器向攻击者指定的非预期目标发起请求，可能用于探测内网、访问内部服务或云元数据接口。
危险等级：高危
OWASP分类：A10:2021 - 服务端请求伪造
常见场景：根据用户输入URL发起请求的功能（如图片抓取、Webhook回调、PDF生成服务）
防御方法：对用户提供的URL进行白名单校验、禁止访问内网地址段、关闭不必要的URL重定向跟随、网络层隔离内部服务
